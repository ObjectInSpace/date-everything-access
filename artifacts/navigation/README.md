# Navigation Asset Findings

This folder contains asset-mined navigation inputs from the ripped `ThirdPersonGreybox.unity` scene.

## Files

- `thirdpersongreybox-navigation-data.json`
  - Generated by `.\scripts\Export-SceneNavigationData.ps1`
  - Includes serialized `CameraSpaces`, door-like scene objects with world positions, camera objects, and teleporter endpoints
- `thirdpersongreybox-blockers.json`
  - Generated by `.\scripts\Export-SceneBlockerData.ps1`
  - Includes primitive collider exports plus a filtered `NavigationBlockers` list for large static walk-blocking footprints
  - Counts mesh and terrain colliders for diagnostics, but does not yet rasterize them into blocker geometry
- `local_navigation_maps.generated.json`
  - Generated by `.\scripts\Build-LocalNavigationMaps.ps1`
  - Includes per-graph-zone occupancy grids built from `CameraSpaces` envelopes minus filtered blocker footprints
  - Copied into `BepInEx\plugins\local_navigation_maps.generated.json` during mod builds and now consumed at runtime for short in-zone waypoint planning
- `navmesh_export.live.json`
  - Generated in-game by pressing `Ctrl+Shift+F9`
  - Copied back into this folder by `.\scripts\Import-NavMeshExport.ps1`
  - Includes the live Unity navmesh triangulation plus per-transition checks against the current `navigation_graph.json`
- `transition_sweep.live.json`
  - Generated in-game by pressing `Ctrl+Alt+Shift+F9`
  - Imported and merged back into this folder by `.\scripts\Import-TransitionSweepReport.ps1`
  - Preserves prior passed open-passage results across reruns and feeds repo-side override refreshes
- `transition_sweep.summary.txt`
  - Generated by `.\scripts\Import-TransitionSweepReport.ps1`
  - Lists merged totals, override refresh results, and the remaining failed open-passage transitions
- `door_transition_sweep.live.json`
  - Generated in-game by pressing `Ctrl+Alt+Shift+F6`
  - Imported and merged back into this folder by `.\scripts\Import-DoorTransitionSweepReport.ps1`
  - Preserves prior passed door results across reruns for repo-side review
- `door_transition_sweep.summary.txt`
  - Generated by `.\scripts\Import-DoorTransitionSweepReport.ps1`
  - Lists merged totals plus the remaining door failures, including connector and interaction clues
- `thirdpersongreybox-navmesh-assets-report.json`
  - Generated by `.\scripts\Inspect-SceneNavMeshAssets.ps1`
  - Summarizes the exported `ThirdPersonGreybox.unity` scene plus the raw Addressables scene bundle
  - Confirms whether the scene actually links a baked `NavMeshData` asset

- A clear current limitation: this runtime pass only uses primitive-blocker occupancy and only plans inside the current source or destination graph zone; mesh and terrain blockers plus object-specific arrival positions are still follow-up work

## Links That Are Probably Open Passages, Not Doors

- `living_room_tutorial -> living_room`
- `hallway -> living_room`
- `living_room -> dining_room`
- `dining_room -> piano_room`
- `dining_room -> kitchen`

These links now derive their waypoints and crossing anchors from the scene's own `CameraSpaces` data instead of from hand-picked room-pair overrides.

Current open-passage build strategy:

- For each graph link marked `OpenPassage`, collect the exact scene zone plus any same-family numbered `CameraSpaces` variants such as `hallway2` to `hallway7`.
- Evaluate every source and destination candidate pair using the serialized zone bounds when present, or the scene point position when the zone has no bounds.
- Pick the pair with the smallest boundary gap, then emit:
  - `FromWaypoint` and `ToWaypoint` from the selected source and destination scene-zone positions
  - `FromCrossingAnchor` and `ToCrossingAnchor` from the closest boundary points between those selected scene zones
- This now covers the six major room-to-room passages and the remaining subzone-to-room open-passage links such as `dorian_office2 -> office` and `kitchen_ronald -> kitchen`.

## Practical Build Order

1. Use teleporter endpoints directly for `office -> crawlspace`.
2. Use paired door cameras for links with obvious `Camera_Dorian...Door...` assets.
3. Use stair cameras for `upper_hallway -> hallway`.
4. Use `CameraSpaces` geometry to derive open-passage anchors, then only add hand-authored overrides if runtime testing proves a specific link still needs them.
